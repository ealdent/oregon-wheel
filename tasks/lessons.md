- Always properly dispose Three.js meshes, materials, and geometries when removing objects from the scene to prevent memory leaks in long-running apps.
- When maintaining a finite number of positions for objects (like task slots), implement an array or stack of available indices and ensure indices are reclaimed when objects are removed.
- For 2D canvas games, cache complex static vector layers (roads, pebbles, grass, decorations) onto an offscreen canvas. Recalculate buffs (like booster towers) statically when towers are placed, upgraded, or sold instead of doing O(N^2) calculations on every physics tick. Render numerous small particles using fillRect rather than beginPath/arc/fill path calls.
- When implementing delta-time (frame-rate independence) in a game loop, ensure that all base speeds, progress increments, particle velocities, and frame-decay timers are scaled appropriately (usually by multiplying by 60 for target 60fps base values) so they do not run 60x slower in real-time. Also, remove heavy CSS features like backdrop-filter from UI panels surrounding the canvas, as they cause massive GPU composite lag on every repaint.
- When drawing real-device training diagrams, verify the physical left/right orientation from the user's viewing angle before placing hotspots. Prefer high-contrast label pills and generous hotspot spacing over decorative schematic labels when the diagram is meant for matching a device in hand.
- Decorative overlays and pseudo-elements in front of interactive UI must use `pointer-events: none` or live behind content with explicit stacking, especially when only one grid column fails hover/click behavior.
